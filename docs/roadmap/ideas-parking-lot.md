## Promotion template

When promoting an idea, create or update a roadmap entry first. Then create a GitHub Issue with:

```markdown
## Objective
## Dependencies
## Product context
## Instructions
## Required behavior
## Acceptance criteria
## Validation
## Non-goals
## Final response requirements
```

The issue body should be directly usable as the implementation prompt. Do not include a separate `Codex prompt` section.
