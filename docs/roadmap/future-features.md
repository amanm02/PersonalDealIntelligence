# Future Feature Backlog

This file stores post-MVP feature ideas that are not part of the active Banking MVP implementation sequence.

Use this backlog to keep `docs/issue-map.md` focused on current execution work.

## Rules

- Do not treat items in this file as active implementation scope.
- Promote a future feature to a GitHub Issue only when it has clear acceptance criteria and validation.
- Keep deferred clothing, travel, flight, hotel, cashback stacking, shopping, browser extension, and dashboard work out of Banking MVP issues.
- Keep safety, privacy, and source-policy constraints consistent with `AGENTS.md` and `docs/architecture/banking-mvp.md`.

## Promotion path

When a future item is ready, create a GitHub Issue with:

```markdown
## Objective
## Dependencies
## Product context
## Instructions
## Required behavior
## Acceptance criteria
## Validation
## Non-goals
## Final response requirements
```

The issue body should be directly usable as the implementation prompt. Do not include a separate implementation-prompt section.
